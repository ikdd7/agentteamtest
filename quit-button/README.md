# 퇴사버튼 🔴 — v0 프로토타입

퇴사하고 싶은 순간, 버튼을 꾹 누르면 전국 카운터가 올라가는 익명 참여형 밈 앱의 UX 프로토타입입니다.

## 실행

의존성 없음. `index.html`을 브라우저로 열면 끝.

```bash
open quit-button/index.html        # 또는 브라우저로 파일 열기
python3 -m http.server -d quit-button 8080   # 서버로 띄우려면
```

모바일 화면(폭 480px 이하) 기준으로 설계됐습니다. 데스크톱에서는 개발자도구 모바일 뷰 권장.

## 구현된 UX (설계 문서의 여정 ①)

| 설계 항목 | 구현 |
|---|---|
| 3초 룰 | 스플래시·가입·팝업 없음, 랜딩 = 버튼 |
| 홀드 투 파이어 | 1.2초 길게 누르기, 진행 링 게이지, 햅틱 램프업(지원 기기) |
| 중도 이탈 연출 | 버튼이 통통 튕겨 올라오며 "아직 아닌가 봐요! 🐣" |
| 발사 카타르시스 | 화면 셰이크 + 컨페티 + 표정 변화 + "N번째 접수" |
| 순번 표현 | 콜드스타트 원칙대로 절대값 대신 "오늘 N번째" 중심 |
| 밈 카드 | 발사 후 자동 생성(canvas), 이미지 저장 / Web Share |
| 사유 피드 | 익명 140자, 댓글 없음, '나도 🔴'만 |
| 자동 마스킹 | 회사명·직급 호칭 감지 시 "(어느 회사)", "(그분)"으로 치환 |
| 나의 기록 | 오늘/누적/연속 출석일 (localStorage, 기기 내 저장) |
| 다회 누름 케어 | 5회 이상이면 "오늘 좀 심하네요 🥺" 카피 전환 |
| 접근성 | 키보드는 Enter 2회 확인 방식으로 대체 |

## 프로토타입 한계 (정직 고지)

- **전국 카운터는 시뮬레이션**입니다(시간대별 가중치 + 랜덤 틱). 실서비스에서는 실카운터로 교체 — 제품 원칙상 가짜 카운터는 금지이며, 데모임을 화면 푸터에 명시했습니다.
- 사유 피드의 예시 글에는 "예시" 라벨이 붙어 있습니다.
- 봉인권(900원 결제)·푸시·업종 통계는 v1 범위로 미구현입니다.
