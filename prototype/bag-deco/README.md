# 🎀 백꾸 스튜디오 (Bag Deco Studio) — 프로토타입

가방꾸미기(백꾸) 트렌드를 겨냥한 **"가상으로 달아보고 → 바로 구매"** 컨셉의
MVP 프로토타입입니다. 시장의 빈틈인 *백꾸 시뮬레이터 + 커머스 연결*을 가장
작게 검증하기 위한 버전입니다.

## 바로 실행하기

설치·빌드 불필요. 외부 라이브러리도 없습니다.

```
prototype/bag-deco/index.html  → 브라우저로 열기
```

또는 로컬 서버로:

```bash
cd prototype/bag-deco
python3 -m http.server 8000
# http://localhost:8000 접속
```

## 비즈니스 모델 — 오늘의집형 디스커버리 커머스

직접 재고·결제를 하지 않습니다. **"꾸미기/영감 → 아이템 정보 → 실제 판매처로
아웃바운드"** 로 트래픽을 일으키고 제휴 수수료로 수익화합니다.

- 콜드스타트 없음(재고·물류·결제 불필요)
- KPI = **아웃바운드 트래픽**(판매처 이동 수), 공유 수
- 각 아이템의 "사러 가기"는 실제 판매처(데모: 네이버쇼핑 검색)로 연결

## 담긴 핵심 기능 (MVP)

| 기능 | 설명 |
|------|------|
| 🎒 가방 선택 / 사진 업로드 | 기본 가방 3종(벡터 일러스트) 또는 **내 가방 사진** 배경 |
| ✨ 아이템 배치 | 키링·참 10종을 탭해서 추가, **드래그·크기·회전** |
| ✨ 둘러보기(갤러리) | 다른 사람 백꾸 구경 → **따라 꾸미기** / 아이템 사러 가기, 내 백꾸 올리기·좋아요 |
| 🧾 아이템 상세 카드 | 브랜드·가격대·재질·설명 (오늘의집 "이 집 아이템" 카드형) |
| 🛒 사러 가기(아웃바운드) | 아이템마다 **실제 판매처로 이동** + 이동 횟수 집계 |
| 📋 쇼핑 리스트 복사·공유 | 꾸민 조합을 링크 리스트로 복사 → 공유가 **또 다른 트래픽** |
| 📸 이미지 저장 | 완성 백꾸를 **PNG로 다운로드** → 인스타/틱톡 공유용 |

## 이 프로토타입이 검증하려는 것

1. 꾸민 사람 중 몇 %가 **"사러 가기"(아웃바운드)** 를 누르나? (핵심 KPI)
2. 저장 이미지·쇼핑 리스트가 **공유(바이럴 루프)** 를 유발하는가?
3. 어떤 아이템/맥락에서 이동률이 높은가? (고가 참 vs 저가 키링)

## 의도적으로 뺀 것 (다음 단계)

- 실제 제휴 링크/딥링크 연동(다이소몰·지그재그·무신사·쿠팡 파트너스)
- 갤러리 서버 저장·실시간 피드 (현재는 세션 내 메모리)
- AI 추천 조합 ("이 가방엔 이 키링")
- 로그인·팔로우·댓글

## 기술 메모

- 순수 HTML/CSS/Vanilla JS — 의존성 0
- 아이템은 이미지 에셋 대신 **이모지**로 렌더 → 오프라인 즉시 동작, 에셋 준비 불필요
- 이미지 저장은 외부 라이브러리(html2canvas) 없이 **Canvas API로 직접 렌더**
- 실제 제품화 시: React Native/Flutter + Supabase, 이모지는 실제 상품 이미지로 교체
