# 🎀 백꾸 스튜디오 (Bag Deco Studio) — 프로토타입

가방꾸미기(백꾸) 트렌드를 겨냥한 **"가상으로 달아보고 → 바로 구매"** 컨셉의
MVP 프로토타입입니다. 시장의 빈틈인 *백꾸 시뮬레이터 + 커머스 연결*을 가장
작게 검증하기 위한 버전입니다.

## 바로 실행하기

설치·빌드 불필요. 외부 라이브러리도 없습니다.

```
prototype/bag-deco/index.html  → 브라우저로 열기
```

또는 로컬 서버로:

```bash
cd prototype/bag-deco
python3 -m http.server 8000
# http://localhost:8000 접속
```

## 담긴 핵심 기능 (MVP)

| 기능 | 설명 |
|------|------|
| 🎒 가방 선택 / 사진 업로드 | 기본 가방 4종 또는 **내 가방 사진**을 배경으로 |
| ✨ 아이템 배치 | 키링·참·스티커 16종을 탭해서 추가, **드래그로 이동** |
| 🔧 크기 / 회전 / 삭제 | 선택한 아이템을 슬라이더로 조절, 맨 앞으로 보내기 |
| 🛍️ 실시간 장바구니 | 올린 아이템이 자동으로 장바구니·합계에 반영 |
| 📸 이미지 저장 | 완성된 백꾸를 **PNG로 다운로드** → 인스타 공유용 |

## 이 프로토타입이 검증하려는 것

1. **사람들이 실제로 "사기 전에 미리 달아보는" 행동을 하는가?** (핵심 가설)
2. 꾸민 결과가 그대로 **장바구니로 이어지는 흐름**이 자연스러운가?
3. 저장 이미지가 **공유(바이럴 루프)** 를 유발하는가?

## 의도적으로 뺀 것 (다음 단계)

- AR 실시간 미리보기 (ARKit/ARCore)
- AI 추천 조합 ("이 가방엔 이 키링")
- 실제 결제 / 작가 마켓플레이스 입점
- 로그인·커뮤니티 피드

> 검증 전 단계에서는 "캔버스 + 공유"가 실제로 쓰이는지부터 확인하는 것이
> 우선이므로 위 기능은 의도적으로 제외했습니다.

## 기술 메모

- 순수 HTML/CSS/Vanilla JS — 의존성 0
- 아이템은 이미지 에셋 대신 **이모지**로 렌더 → 오프라인 즉시 동작, 에셋 준비 불필요
- 이미지 저장은 외부 라이브러리(html2canvas) 없이 **Canvas API로 직접 렌더**
- 실제 제품화 시: React Native/Flutter + Supabase, 이모지는 실제 상품 이미지로 교체
