# PRD — 느좋점수 (NeujotScore)

> **한 줄 정의**: 사진을 올리면 사람들이 "느좋(느낌 좋은) 점수"와 무드 태그로 평가해주고,
> 더 느좋해지는 스타일 피드백까지 받는 **감성·스타일 평가 놀이터**.
> 핵심은 "얼굴 우열"이 아니라 **분위기·무드·코디**를 본다는 것.

- 문서 버전: v0.1 (초안)
- 작성일: 2026-06-18
- 상태: 기획 확정 → MVP 설계
- 타깃 출시: MVP 4~6주 (1인 바이브코딩 기준)

---

## 0. 왜 이걸 만드는가 (배경 / 검증 요약)

이 아이디어는 여러 후보를 거른 끝에 다음 조건을 동시에 만족해서 선택됐다.

| 기준 | 충족 여부 |
|------|-----------|
| 재미·콘텐츠형 (지루한 트래커 아님) | ✅ |
| 한국인의 "외모·감성" 관심사와 직결 | ✅ |
| 바이럴 연료 내장 ("느좋"은 이미 Z세대 유행어) | ✅ |
| 리텐션 구조 (자기 평가받기 욕구 + 상호 게이팅) | ✅ |
| AI 얼평 레드오션과 차별 (사람 평가 + 무드 프레이밍) | ✅ |
| 저작권/콜드스타트/독성 함정 회피 설계 가능 | ✅ (아래 §7, §8) |

### 차별점 핵심
- **AI가 아니라 진짜 사람들의 평가** (AI 얼평/VibeRate와 다름)
- **얼굴 점수가 아니라 "느좋(무드·스타일·분위기)" 점수** → 덜 잔인하고, "바꿀 수 있는" 긍정 피드백
- **"느좋"이라는 유행어 자체가 마케팅** (느좋남/느좋녀/느좋코디 트렌드 탑승)

## 1. 타깃 사용자

### 1차 타깃
- **18~29세, 인스타·트위터에서 "느좋" 감성을 소비/생산하는 Z세대**
- 자기 사진/코디를 올리고 반응을 얻고 싶지만, 인스타는 부담스럽고 솔직한 피드백이 없음

### 2차 타깃
- 소개팅/프로필 사진을 고르려는 사람 ("어떤 게 더 느좋?")
- 코디·OOTD에 진심인 사람 (스타일 피드백 수요)

### 페르소나
> "민지(24)는 데이트룩 사진 2장 중 뭘 올릴지 고민이다. 친구한테 물으면 다
> 예쁘다고만 한다. **솔직하고 익명인, 그러나 잔인하지 않은** 반응이 필요하다."

## 2. 핵심 가치 제안 (Value Proposition)

| 사용자 고통 | 느좋점수의 해결 |
|-------------|-----------------|
| 내 사진/스타일이 객관적으로 어떤 느낌인지 모름 | 익명 다수의 **느좋점수 + 무드 태그** |
| 인스타 좋아요는 의례적, 솔직하지 않음 | **솔직하되 잔인하지 않은(무드 기반)** 피드백 |
| 외모 평가 앱은 상처를 줌 | "얼굴"이 아니라 **"무드/코디" 평가 → 개선 가능** |
| AI 점수는 영혼 없음 | **사람의 진짜 감각** |

## 3. 핵심 루프 (제품의 심장)

```
        ┌─────────────────────────────────────────────┐
        │                                             │
        ▼                                             │
  [사진 업로드]                                        │
        │                                             │
        ▼                                             │
  [AI 즉석 느좋 추정] ← 콜드스타트 해결, 0명이어도 결과   │
        │                                             │
        ▼                                             │
  [다른 유저들이 느좋점수 + 무드 태그 평가]               │
        │                                             │
        ▼                                             │
  [결과 카드 생성 → 공유(카톡/인스타 스토리)] ← 바이럴      │
        │                                             │
        ▼                                             │
  [재평가 받고 싶다 / 새 사진 올리고 싶다]                │
        │                                             │
        ▼                                             │
  [★게이팅: 남 사진 10개 평가해야 가능] ──────────────────┘
        (= 평가 공급 확보 + 리텐션 + 유동성)
```

### 게이팅 규칙 (Photofeeler 카르마 모델 응용)
- **첫 사진 1장**: 게이팅 없이 바로 업로드·평가받기 (온보딩 마찰 최소화)
- **삭제 후 재평가 / 2번째 사진부터**: **남의 사진 10개를 평가해야 1회 가능**
- 평가 크레딧을 "느좋 캔디🍬" 같은 재화로 시각화 → 모으는 재미
- (수익화) 캔디를 결제로도 충전 가능 → 평가 안 하고 바로 받기

## 4. 평가 메커니즘 설계 (⭐ 독성 회피의 핵심)

> **원칙: 사람(얼굴)을 채점하지 않는다. "사진이 주는 무드/스타일"을 평가한다.**

### 4-1. 점수 입력 방식
- 한 줄 잔인한 숫자(1~10 외모점수) **금지**.
- 대신 **"느좋 슬라이더"** + **무드 태그 다중선택**:

**느좋 슬라이더** (단일 축, 긍정 프레이밍)
```
영혼없음 ──────●─────── 느좋 폭발
        (이 사진의 전체적인 무드가 얼마나 느좋?)
```

**무드 태그** (복수 선택, 전부 긍정/중립 — 비방 불가)
- 분위기: `청량한` `시크한` `따뜻한` `빈티지` `미니멀` `힙한` `러블리` `차분한`
- 스타일: `코디 느좋` `색감 깡패` `구도 좋음` `포즈 자연스러움`
- 종합: `프사각` `소개팅각` `인스타 메인각`

### 4-2. 결과 표시
- **느좋점수**: 0~100 (다수 슬라이더 평균을 환산)
- **무드 프로필**: 가장 많이 받은 태그 TOP 3 ("당신의 무드: 시크 · 빈티지 · 색감깡패")
- **개선 코칭(AI)**: "조명이 한 톤만 따뜻하면 느좋지수 +α" 같은 **실행가능·긍정** 제안

### 4-3. 절대 하지 않는 것 (안전 가드레일)
- ❌ "못생김/노안/비호감" 같은 **부정 태그 없음**
- ❌ 사용자 간 **외모 순위표(랭킹) 없음** (자기 점수 추이만 보여줌)
- ❌ 자유서술 욕설 입력 없음 (태그·슬라이더 구조화 입력만)

## 5. 기능 명세

### 5-1. MVP (must-have)
| # | 기능 | 설명 |
|---|------|------|
| M1 | 사진 업로드 | 1장, 자동 EXIF 제거, 얼굴 외 배경 포함 무드 평가 |
| M2 | AI 즉석 느좋 추정 | 업로드 즉시 임시 점수+태그 (콜드스타트 방지) |
| M3 | 평가 화면 | 슬라이더 + 무드 태그, 스와이프로 다음 사진 |
| M4 | 게이팅(캔디) | 10개 평가 = 재평가권 1개 |
| M5 | 결과 카드 | 느좋점수 + 무드 TOP3 + 공유 이미지 자동 생성 |
| M6 | 공유 | 카톡/인스타 스토리 규격 이미지 다운로드·공유 |
| M7 | 신고/차단 | 부적절 사진 신고, 자동 블러 후 검수 |
| M8 | 연령 게이트 | 만 14세 미만 차단 (안전) |

### 5-2. v1 (should-have)
- 사진 2장 A/B 비교 ("뭐가 더 느좋?")
- 내 느좋점수 추이 그래프 (자기 성장만)
- 무드별 피드 ("오늘의 빈티지 느좋 사진들") = 콘텐츠 커뮤니티화
- 친구 초대 → 캔디 보상

### 5-3. v2 (could-have)
- 코디/소개팅/프사 등 **목적별 평가 모드**
- "느좋 크리에이터" 랭킹(긍정 한정) · 배지
- 브랜드/쇼핑 제휴 ("이 무드엔 이 아이템")

## 6. 바이럴 & 리텐션 설계

### 바이럴 (획득)
- **공유 카드가 곧 광고**: "내 무드: 시크·빈티지 / 느좋점수 87 🔥" → 스토리 공유 자연 유입
- "느좋"이라는 **검색·해시태그 트렌드** 올라타기
- A/B 비교 결과는 공유 유인이 특히 큼 ("1번이 이김 ㅋㅋ")

### 리텐션 (유지)
- **자기 점수 확인 욕구** = 핵심 복귀 동기 (새 평가 알림 푸시)
- **게이팅**이 평가 행동을 습관화
- **데일리 무드 챌린지** ("오늘의 무드: 청량 — 청량 느좋 사진 올려봐")
- 무드 프로필 변화 추적 ("지난주보다 '힙한' 태그 늘었어요")

## 7. 안전 · 모더레이션 · 법무 (가볍게 보면 망하는 영역)

| 영역 | 리스크 | 대응 |
|------|--------|------|
| 미성년 보호 | 청소년 외모평가 폐해 | 14세 미만 차단, 외모 아닌 무드 프레이밍, 부정 피드백 구조적 차단 |
| 음란/불쾌 사진 | NSFW 업로드 | 업로드 시 AI NSFW 필터 → 의심 시 보류·검수, 유저 신고 |
| 초상권 | 타인 사진 무단 업로드 | "본인 사진만" 약관 + 신고 시 즉시 삭제, 얼굴 타인 다수 감지 시 경고 |
| 개인정보 | 사진 보관 | EXIF/위치 제거, 삭제 시 즉시 파기, 평가 종료 후 노출 제한 |
| 정신건강 | 점수 집착 | 순위표 없음·부정태그 없음·긍정 코칭, 점수 낮을 때 격려 카피 |

> **포지셔닝 자체가 안전장치**: "외모 채점기"가 아니라 "무드/스타일 큐레이션 놀이"로
> 일관되게 커뮤니케이션. 마케팅·UX 카피에서 "예쁘다/잘생김" 어휘를 피하고
> "느좋/무드/감성/분위기"로 통일.

## 8. 콜드스타트 전략 (초기 평가 공급 문제)

신규 서비스 최대 난관 = "올렸는데 평가가 없다".

1. **AI 즉석 추정** (M2): 사람 0명이어도 즉각 결과 → 빈손 경험 제거
2. **시드 평가단**: 초기엔 운영자/지인이 평가 큐 채움
3. **게이팅으로 공급 자급**: 받으려면 평가해야 하므로 사용자 증가 = 평가 공급 증가
4. **타깃 커뮤니티 침투**: 트위터 느좋 태그·대학 에타·인스타에서 시드 유저 확보

## 10. 화면 흐름 (MVP)

```
온보딩(연령·약관)
  → 첫 사진 업로드
  → AI 즉석 느좋 결과 (맛보기)
  → "더 많은 사람의 평가 받기" → 평가하러 가기(게이팅 안내)
  → 평가 화면(슬라이더+태그, 10장 스와이프)
  → 캔디 획득 → 내 사진 평가 열림
  → 결과 카드 → 공유
  → 푸시: "새 평가 3개 도착!" → 복귀
```

## 11. 기술 스택 (1인 바이브코딩 기준)

> 원칙: 빠르게·적은 운영부담·확장 가능.

- **프론트**: Next.js (웹 우선, PWA로 앱 느낌) — 앱스토어 심사 없이 빠른 반복
- **백엔드/DB/인증/스토리지**: Supabase (Postgres + Storage + Auth 한 번에)
- **이미지 처리**: 클라이언트 EXIF 제거 + 서버 리사이즈
- **AI 즉석 추정 / NSFW 필터 / 무드 태그 추천**: 비전 모델 API (이미지→무드 태그·점수)
- **공유 카드 생성**: 서버사이드 OG 이미지 생성 (Satori/`@vercel/og`)
- **푸시**: 웹푸시(FCM)
- **배포**: Vercel + Supabase (둘 다 무료 티어로 MVP 가능)

### 데이터 모델 (초안)
```
users(id, age_verified, candy, created_at)
photos(id, user_id, url, status[active|deleted|hidden], ai_score, created_at)
ratings(id, photo_id, rater_id, vibe_score, tags[], created_at)
tags(id, name, category, polarity=positive)   # 부정 태그 없음
reports(id, photo_id, reporter_id, reason, created_at)
```
