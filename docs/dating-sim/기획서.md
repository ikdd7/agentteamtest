# 연애 시뮬레이션 게임 기획서 (v0.1)

> 가제: **「설렘 100%」** (working title)
> 한 줄 컨셉: *카톡으로 호감도를 쌓고, 데이트와 설렘 이벤트로 그 사람과 사귀는, 고등학교 3년 청춘 연애 시뮬레이션*

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 연애 시뮬레이션 / 비주얼 노벨 + 메신저(채팅) 인터랙션 |
| 플랫폼 | 모바일 우선(세로 화면), 웹(PWA)으로 먼저 프로토타이핑 |
| 타깃 | 10대~20대 초반, *딸기100%* 같은 청춘 로맨스 감성을 좋아하는 층 (특히 여성향) |
| 핵심 무드 | 풋풋함, 두근거림, 첫사랑, 교실/방과후/계절감, 살짝의 삼각관계 긴장감 |
| 플레이 시간 | 1회차(한 명 공략) 약 2~4시간, 멀티 엔딩으로 반복 플레이 |
| 시점/주인공 | 여고생 주인공(이름·외모 커스터마이즈 가능), 1인칭 감정선 |
| 목표 | 입학~졸업 3년 동안 마음에 드는 상대의 호감도를 키워 **고백 성공 → 연애 → 졸업 엔딩** |

### 레퍼런스 톤 (*딸기100%* 여성판 느낌)
- 우연한 만남(옥상/도서관/비 오는 날)에서 시작되는 설렘
- 여러 매력적인 남자 캐릭터 사이에서 흔들리는 마음
- "둘 중 누구?" 식의 삼각관계 긴장과 질투 이벤트
- 교복, 축제, 수학여행, 졸업식 등 한국 고등학교 시즌 이벤트

## 2. 핵심 재미 (Core Loop)

```
        ┌─────────────────────────────────────────────┐
        │  하루(턴) 시작 → 행동 선택(어디서 누구와)         │
        │      │                                        │
        │      ▼                                        │
        │  카톡 대화: 답장 선택지로 호감도 ±               │
        │      │                                        │
        │      ▼                                        │
        │  조건 충족 시 → 데이트/설렘 이벤트 발생(CG 연출)   │
        │      │                                        │
        │      ▼                                        │
        │  호감도·관계 단계 상승 → 다음 시즌 이벤트 해금     │
        └─────────────────────────────────────────────┘
        반복하며 3년(학기/시즌) 진행 → 고백 분기 → 엔딩
```

핵심은 **"카톡 대화로 호감도를 올린다"** 라는 한 가지 동사를 가장 재밌게 만드는 것.
- 짧고 가벼운 텍스트, 이모티콘, 읽씹/안읽씹, 답장 타이밍 같은 *진짜 메신저* 디테일로 몰입.
- 선택지마다 상대 성격에 따라 반응이 갈려서 "이 사람은 이런 말에 약하구나"를 학습하는 재미.

## 3. 호감도 / 관계 시스템

### 3.1 수치
- **호감도(Affection) 0~100**: 캐릭터별로 누적. 대화·이벤트·선물로 변동.
- **관계 단계(Stage)**: 호감도 구간에 따라 변함
  | 단계 | 호감도 | 설명 |
  |------|--------|------|
  | 남남 | 0~19 | 그냥 같은 반/아는 사이 |
  | 친구 | 20~44 | 편하게 카톡하는 사이 |
  | 썸 | 45~69 | 둘 다 의식하기 시작, 설렘 이벤트 해금 |
  | 고백 가능 | 70~89 | 고백 분기 진입 가능 |
  | 연인 | 90~100 | 사귀는 중, 커플 전용 이벤트 |
- **숨은 호감도(Hidden)**: 화면엔 안 보이고, 결정적 분기/질투 이벤트에만 영향(긴장감용).

### 3.2 변동 규칙
- 대화 선택지: 상대 *성격/취향 태그*와 매칭되면 +, 어긋나면 − 또는 무반응(읽씹).
- 같은 말 반복/과한 텍스트 폭탄 → 호감도 정체 또는 소폭 감소(현실감).
- 기념일·생일 챙기기, 적절한 선물 → 보너스.
- 다른 캐릭터에게 너무 치우치면 → 질투/서운함 이벤트로 일시 하락.

## 4. 카톡형 대화 시스템 (게임의 심장)

화면은 **실제 메신저 UI**를 모방:
- 말풍선(좌: 상대 / 우: 나), 프로필 사진, 시간, 읽음 표시(1 사라짐).
- 입력 시 **2~4개의 답장 선택지**가 뜸 → 고르면 내 말풍선 전송.
- 상대가 "입력 중…" 인디케이터 후 답장(타이핑 연출로 두근거림 연출).
- **이모티콘/스티커** 선택지, 사진 전송(이벤트 트리거).
- **안읽씹/읽씹 메커닉**: 특정 상황에서 상대가 일부러 답을 늦게 → 플레이어 애태우기.

### 대화 데이터 구조(개념)
```jsonc
{
  "id": "min_ch1_rain",
  "character": "minjun",
  "trigger": { "season": "1-1", "affection_min": 20 },
  "lines": [
    { "from": "minjun", "text": "야 우산 있어? ㅋㅋ", "delay": 1.2 },
    {
      "from": "me",
      "choices": [
        { "text": "어 같이 쓸래?", "affection": +5, "tag": "적극" },
        { "text": "없는데… 어떡하지", "affection": +2, "tag": "소심" },
        { "text": "너 우산 뺏어야지 ㅋㅋ", "affection": -1, "tag": "장난", "char_pref": "tsundere:+3" }
      ]
    }
  ],
  "reward_event": "umbrella_share"  // 조건 충족 시 데이트/CG 이벤트로 연결
}
```

## 5. 데이트 / 설렘 이벤트 시스템

- **설렘 이벤트(Heart Event)**: 썸 단계 이상에서 해금. 일러스트(CG) + 짧은 비주얼 노벨 연출.
  예) 같이 우산 쓰기, 옥상에서 단둘이, 축제 야간 불꽃놀이, 손 잡기, 첫 데이트, 고백.
- **데이트**: 주말 턴에 장소 선택(영화관·카페·바다·놀이공원). 장소마다 대화·미니 선택지 → 호감도 큰 폭 상승 + 추억(앨범) 획득.
- **CG 앨범 / 추억 수집**: 달성한 설렘 이벤트의 일러스트를 갤러리에 보관(수집 욕구·반복 플레이 동기).
- **분기 이벤트**: 같은 시간대에 두 캐릭터가 동시에 부르면 *누구를 선택* → 삼각관계 긴장.

## 6. 3년(입학~졸업) 진행 구조

시간 단위 = **시즌(학기)**. 총 3학년 × 약 6시즌 = 18시즌 + 주요 학사 이벤트.

| 학년 | 시즌/이벤트 | 연애 단계 목표 |
|------|------------|----------------|
| 1학년 | 입학식 · 반 배정 · 중간고사 · 여름방학 · 체육대회 · 겨울방학 | 캐릭터 만남, 친구 되기(호감도 ~40) |
| 2학년 | 새 반 · 수학여행 · 축제 · 비 오는 장마 · 크리스마스 | 썸 ~ 첫 설렘 이벤트(호감도 ~70) |
| 3학년 | 수능 압박 · 마지막 축제 · 졸업여행 · 졸업식 · 화이트데이 | 고백 → 연인 → 졸업 엔딩(호감도 90+) |

- 각 시즌에 **공통 학사 이벤트**(누구와 플레이하든 발생)와 **캐릭터 전용 이벤트**가 섞임.
- "수능"처럼 살짝의 성장/일상 요소(공부·동아리)도 가볍게 넣어 청춘 서사 강화(과하지 않게).

## 8. 화면 구성 (UI 흐름)

```
[타이틀] → [새 게임/이어하기/갤러리]
   │
   ▼
[프롤로그·주인공 설정] → [홈(달력/오늘의 행동)]
   │            ┌──────────────┬──────────────┬─────────┐
   ▼            ▼              ▼              ▼         ▼
[카톡 목록] [장소 이동/데이트] [캐릭터 상태(호감도)] [갤러리] [설정]
   │
   ▼
[채팅방(대화·선택지)] →(조건)→ [설렘 이벤트/CG 비주얼노벨]
```

핵심 화면 3개만 먼저: **① 홈(달력) ② 카톡 채팅방 ③ 이벤트(CG+대사)**.

## 9. 기술 스택 (제안)

프로토타입을 빠르게 만들고 모바일까지 가는 경로 기준 **추천안**:

- **프론트엔드**: React + TypeScript + Vite (웹/PWA → 추후 Capacitor로 앱 패키징)
- **상태관리**: Zustand (가벼움) — 호감도/플래그/세이브
- **스타일**: Tailwind CSS (메신저 UI 빠르게)
- **데이터**: 시나리오/대사를 **JSON(또는 YAML)** 데이터로 분리 → 기획자가 코드 없이 대사 추가
- **세이브**: localStorage(웹) → 추후 클라우드
- **에셋**: 캐릭터 일러스트/CG는 초기엔 플레이스홀더(도형/AI 임시) → 후속 제작

> 대안: 비주얼 노벨 전용 엔진 **Ren'Py**(파이썬)도 강력하지만, 카톡 UI/모바일 감성 커스터마이즈와 웹 배포 자유도는 React 쪽이 유리. 카톡형 인터랙션이 핵심이라 **React안을 1순위 추천.**

## 10. 데이터 모델 (초안)

```ts
type Character = {
  id: string
  name: string
  archetype: string
  personalityTags: string[]   // 예: ["소꿉친구","햇살"]
  prefTags: string[]          // 좋아하는 화제/말투
  affection: number           // 0~100
  hidden: number              // 숨은 호감도
  unlockedEvents: string[]
}

type GameState = {
  player: { name: string; avatar: string }
  day: number                 // 전체 진행 일/턴
  season: string              // "1-1" ... "3-6"
  characters: Record<string, Character>
  flags: Record<string, boolean>   // 이벤트 분기 플래그
  album: string[]             // 획득한 CG id
  currentRoute?: string       // 메인 공략 대상
}
```
