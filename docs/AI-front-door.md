# AI Front Door (가칭)

> **"나에게 연락·지원·문의하려면 반드시 거쳐야 하는, 나를 학습한 AI 관문."**
> 방문자는 내 AI와 대화해 답을 얻거나 용건을 남기고, 모든 리드·문의가 내 인박스에 쌓인다.

## 왜 K≥1이 가능한가 — 강제 노출 루프(핵심)
- **"내 연락처는 이 링크 하나"** → 나에게 닿으려는 사람은 *반드시* 내 AI를 쓴다(선택이 아니라 필수). Calendly가 '미팅 잡기'를 강제화했듯, 우리는 **'연락·문의'를 강제화**한다.
- 답변 푸터에 **"Powered by · 나도 내 관문 만들기"** → 방문자(특히 동종 프리랜서/구직자)가 자기 것 생성. **사용=노출이 기능에 내장**(Loom 워터마크 패턴) → K를 0.5 → 1 근처로.

## 싱글플레이어 가치 (day 1, 네트워크 0명이어도)
프리랜서/구직자/세일즈: **"24시간 일하는 내 비서"** — 잠재고객·채용담당 질문에 자동 응대 + 리드 수집. 혼자 써도 즉시 유용 = 콜드스타트 없음.

## 끈적함 (리텐션 앵커 = 챗이 아니라 인박스)
매일 **"누가 뭘 물었나 / 새 리드·면접 제안"** 알림 → 돌아올 이유. (가치 수령자를 방문자→소유자로 이동 = set-and-forget 깨기.)

## MVP — 강제 루프를 만드는 최소 (그 이상 금지)
1. 입력 1개(이력서/LinkedIn/사이트 URL) → 나를 학습한 AI.
2. **공유 링크 1개**(= 내 유일한 연락 관문). 방문자: 대화 + **"용건 남기기"**(이름·연락처·메시지).
3. 소유자 **인박스 + 일일 요약 알림** + (옵션)자동 응대 초안.
4. 답변 푸터 **attribution + "Make your own"**.
→ 결제·아바타·팀·분석은 전부 나중.

## 첫 1000명
- 빌더 본인 **도그푸딩** → **빌드인퍼블릭**(X / Indie Hackers / Product Hunt).
- 세그먼트: **구직 개발자·디자이너**(속도) → **프리랜서**(지속성). 채널: r/cscareerquestions · r/forhire · LinkedIn #OpenToWork · 부트캠프 디스코드.
- **손으로 멋진 시드 20~50명 초청** — 그들의 링크가 곧 광고판.

## 규모 시 네트워크 효과 (해자)
페르소나·소개 그래프: "이 분야는 X에게" 라우팅, 사람↔사람 연결. 질의 데이터가 쌓일수록 매칭·응대 품질↑ → 대화형 링크드인 잠재력.

## 바이브코딩 스택
Next.js + 서버리스 API · LLM(요약·RAG·응대) · 임베드 위젯 · Postgres(Supabase) · 이메일 알림(Resend). 본질은 **"입력 파서 + RAG + 공유 위젯 + 인박스"** 얇은 조합 → 주말에 골격.

## #1 리스크 + 1주 검증 실험
- **리스크:** "내 연락처를 AI 관문 하나로 강제"하는 행동을 사람들이 실제로 받아들이나(특히 구직자→채용담당).
- **실험:** 빌더 본인 + 시드 10명이 관문 링크를 실제 바이오·이메일 서명에 걸고 → **방문자 대화율 · 용건 남김률 · "Make your own" 클릭률** 측정 → K 가늠. (합격선: 방문자 대화율 ≥40%, make-your-own 클릭 ≥2%.)

## 포지셔닝 한 줄
"Calendly가 미팅 잡기를 자동화했듯, 우리는 **'당신에 대한 질문 응대'를 자동화**한다." — "프로필/아바타"(Linktree·HeyGen)가 아니라 **"24시간 영업·구직 비서"**(아웃컴)로.
