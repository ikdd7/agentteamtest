## #1 — AI 빌드 여정·실패 로그 (Anti-Recipe / 삽질 일지)
*제안: 2026-06-14 · 렌즈 출처: futurist + 대다모 엔진 이식*

- **한줄:** AI 도구로 빌드하다 막힌 순간(실패·함정·우회)을 도구·버전·증상 단위로 올리는 "삽질 일지" 네트워크. 단발 제보가 아니라 *진행형 빌드 여정*으로.
- 공급=욕구 **14** (디버깅으로 피 본 사람의 이타심+선발자 과시) / 수요 **14** (에러 메시지=검색어) / 콜드스타트 **13** (창업자가 도구 10개 굴려 시드, 본인이 당사자) / 경쟁 약함 **13** (마켓플레이스는 "되는 것"만, "안 되는 것"은 빔) / LTV·그릇 **11** (B2B 벤더 모니터링 의존) / AI 방어 **9** (버전 휘발성·컷오프) / 규제 안전 **8** (도구 동작이라 명예훼손 회피, 단 벤더 적대) / 1인 빌드 **5**
- **총점: 87/100**
- 결정적 약점: $20M이 B2B 수익층에 의존 + 도구 벤더 적대 가능 + 가짜 제보 검증.
- 다음 검증: 에러메시지 검색량 실측, "실패를 창피해서 안 올림" 리스크 UX 테스트.

## #2 — 펫푸드 성분 분석 + 급여 후기 ("화해의 펫푸드판")
*제안: 2026-06-14 · 렌즈: 화해(화장품 성분 불안+객관DB+리뷰) 엔진을 펫푸드로 이식*

- **한줄:** 사료·간식·영양제의 성분을 객관 분석해 보여주고, "우리 아이 급여 후기(알러지·기호성·변화)"를 개체 단위로 쌓는 펫푸드 성분·후기 DB.
- 공급=욕구 **14** (내 아이 먹는 것에 극도 예민 → 애착+불안+남돕기로 자발 후기) / 수요 **13** ("○○사료 후기"·"그레인프리 알러지" 검색 폭발) / 콜드스타트 **14** (성분표는 공개정보 → 1인이 인기 제품 300개 성분 카탈로그 시딩) / 경쟁 약함 **12** (네이버 카페·펫프렌즈 리뷰 산발, *구조화된 성분 분석 DB는 빔*; 화해는 사람 화장품만) / LTV·그릇 **13** (평생 반복구매 + 펫시장 급성장 + 어필리에이트 깊음) / AI 방어 **9** (실제 급여 반응=1차 데이터) / 규제 안전 **8** (사료는 의료광고법 직접 적용 약함, 단 "질병 치료 효과" 표방 주의) / 1인 빌드 **5**
- **총점: 88/100**
- 결정적 약점: 펫프렌즈·어바웃펫 등 커머스가 리뷰를 인접 보유 → 차별점은 "커머스 무관 중립 성분DB". 효능 표방 규제선 관리 필요.
- 다음 검증: 펫푸드 후기 검색량 + 기존 카페 글의 구조화 결핍 실측.

## #3 — 공연 관극 기록 네트워크 ("뮤지컬·연극판 Letterboxd")
*제안: 2026-06-14 · 렌즈: Letterboxd(기록·정체성) + 회전문 관객의 N회차 과시 욕구*

- **한줄:** 뮤지컬·연극을 *공연 회차·캐스트 단위*로 기록하고 별점·관극평을 쌓는 덕후 정체성 네트워크. "내가 이 작품 N회차, 이 캐스트로 봤다"를 박제·자랑.
- 공급=욕구 **14** (회전문 관객의 N회차·캐스트별 기록은 병적으로 강하고 트위터 공유 문화 이미 존재) / 수요 **12** (캐스트별 후기 검색, 단 영화보다 좁음) / 콜드스타트 **14** (공연·캐스트 정보 공개 → 1인이 카탈로그 시딩 쉬움) / 경쟁 약함 **13** (플레이DB=정보, 인터파크=예매뿐 — *개인 관극 기록·정체성 네트워크는 진짜 빔*) / LTV·그릇 **11** (객단가 높으나 국내 시장 영화보다 작음 → 브로드웨이·웨스트엔드·일본 2.5차원 글로벌 확장 필요) / AI 방어 **9** (관극 취향·캐스트 평=1차 데이터) / 규제 안전 **10** / 1인 빌드 **5**
- **총점: 88/100**
- 결정적 약점: 국내 단독 시장이 $20M에 빠듯 → 글로벌 공연덕 시장으로 확장해야. 캐스트/회차 데이터 정규화 노가다.
- 다음 검증: 회전문 관객 규모·트위터 관극 기록 행동 빈도 실측, 글로벌(일본 2.5차원) 동일 욕구 확인.

## #4 — 낚시 포인트·조황·물때 기록 네트워크 (거지맵의 형제, 단 지갑이 깊음)
*제안: 2026-06-14 · 렌즈: 극단 니치 욕구(낚시꾼의 조황 자랑·기록 강박)*

- **한줄:** 낚시 포인트×어종×물때×조황을 개체 단위로 기록·공유하는 낚시꾼 전용 네트워크. "오늘 여기서 이거 잡았다"를 박제·자랑하고, 물때·포인트 데이터가 쌓일수록 정확해짐.
- 공급=욕구 **13** (조황 자랑·조행기 기록은 낚시꾼 본능 — 단 *포인트는 비밀로 하려는 심리*가 일부 공급 저해) / 수요 **13** ("○○갯바위 조황"·물때·포인트 검색 강함) / 콜드스타트 **13** (포인트·어종·물때표는 공개정보 → 1인이 카탈로그 시딩) / 경쟁 약함 **13** (낚시 카페·유튜브·물때앱 산발 — *포인트×조황×물때 구조화 DB는 약함*) / LTV·그릇 **12** (낚시 장비·미끼·선상낚시 예약 = 깊은 지갑, 시장 큼) / AI 방어 **9** (실시간 조황=1차 데이터) / 규제 안전 **9** (안전, 단 포인트 남획 논란 소지) / 1인 빌드 **5**
- **총점: 87/100**
- 결정적 약점: "포인트 비밀주의"가 공급의 양날(자랑욕 vs 독점욕). 조황 허위 제보. → "정확한 좌표 대신 권역 단위 + 조황 인증(사진)"으로 완화.
- 메타 교훈: 거지맵류 *지도 니치*는 욕구·시딩 최강이나 보통 **수익 그릇**에서 깎임. 낚시는 그중 *지갑이 깊어* 그릇을 방어한 케이스 → "니치 욕구 × 깊은 지갑" 교집합이 표적.
