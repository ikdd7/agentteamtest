# PRD — Agent Checkout Index

> AI 쇼핑 에이전트가 **실제로 결제를 완주하는지** 측정하는 **중립·멀티에이전트
> 인덱스 + 공개 리더보드 + 머천트 진단 SaaS.**
> 이 문서는 에이전트 팀의 시장조사 → 아이디어 발굴 → 전 직군 95점 검증을 거쳐
> 수렴한 결과물이다. 빌드 전 **14일 검증 통과**가 진행 조건이다.

## 1. 한 줄 정의
여러 AI 쇼핑 에이전트(ChatGPT·Gemini·Perplexity·Claude·오픈소스)로 쇼핑몰
체크아웃을 매일 주행해, "어느 사이트가 에이전트 결제에서 깨지나"를 측정·공개·진단한다.

## 2. 문제 (Why)
- agentic commerce 프로토콜(OpenAI·Stripe **ACP**, Google·Shopify **UCP**, Mastercard
  **Agent Pay**)이 2025~26 등장했으나, **대다수 사이트가 에이전트 결제를 완주 못 한다.**
- 근거(에이전트 팀 리서치): WebArena 단일 태스크 완주 14~62%, 프로덕션 에이전트
  실패율 70~95%(Fiddler), Stripe도 "businesses still face a heavy integration lift" 인정.
- 머천트는 "내가 ChatGPT/Gemini 결제에서 어디서 깨지나"를 모르고, 에이전트
  개발자는 "어느 몰로 보내야 결제가 되나"를 모른다. **그 사이가 비어 있다.**

## 3. 왜 지금 (Timing)
프로토콜 "표준 발표"와 "실제 작동" 사이의 간극이 **지금 최대**. 표준이 통일돼도
가격·재고·인증·배송옵션·에러처리에서 완주율 편차는 남으므로 측정 면적은 유지된다.

## 4. 타깃 & 가치
| 측 | 누구 | 가치 |
|----|------|------|
| 머천트(1차) | Shopify Plus·DTC 브랜드·이커머스 에이전시 | "당신은 N단계에서 깨진다 + 경쟁사 대비 + **에이전트 전환 손실 $추정** + 수정 후 재측정" |
| 에이전트 개발자(2차) | AI 쇼핑 에이전트사·browser-agent 개발자 | "이 몰 통과율 92%" 라우팅/완주율 데이터 API |

## 5. 해자 (전 직군이 합의한 방어선)
1. **멀티에이전트 중립성** — 여러 에이전트의 end-to-end 완주율을 중립 측정. **어떤
   단일 당사자(Stripe/Google/Visa)도 줄 인센티브가 없는** 유일한 데이터 = 흡수 불가.
2. **데이터 플라이휠** — 값싼 추론으로 매일 측정 → 시계열 통과율 인덱스가 시간이
   갈수록 두꺼워짐(소급 생성 불가).
3. **리더보드 권위** — 공개 "Agent-Readiness 리더보드"를 6개월 내 선점 = 인용 표준.
4. **타이밍** — 위 3장.

## 6. 측정 설계 (법적 안전 — 핵심)
> 보안 직군이 원안을 **22/100**으로 평가한 이유 = 무동의 봇 체크아웃 대량 주행은
> CFAA(상태변경 트랜잭션은 hiQ 보호 밖)·ToS 위반·카드테스팅 분류. 아래로 해소.

- **결제 폼 도달까지만 측정. 결제 제출/실거래/환불 경로 없음**(= `validation/`의 퍼널과 동일).
- **지표 2계층 분리**: `reachability`(결제 폼 도달, 비-트랜잭션) / `completion`(실완주,
  **동의·샌드박스 머천트 한정**).
- **"봇 차단(blocked)"과 "체크아웃 실패(failed)" 분리** — 측정 불가와 실제 결함 구분.
- **증거 의무화**(스크린샷 + DOM + 응답코드), 2-runner 교차검증으로 귀속 신뢰도 확보.
- **방법론 오픈소스 공개**(닐슨 시청률처럼 방법론 자체가 표준이 되게).
- 공개 리더보드는 익명·집계 도달성, 심층/완주율은 동의 머천트만.

## 7. MVP 범위
**반드시:** 멀티에이전트 reachability 측정(50~200 사이트, 일 1회), 차단/실패 분리,
증거 필드, 시계열 DB, 공개 리더보드, 머천트 무료 진단 1건→유료 전환.
**자를 것:** 실결제 완주(동의 머천트 전까지), 에이전트 API 과금(2차), 수만 사이트
스케일, 로그인 필요 사이트, 다지역/통화.

## 8. 기술 스택 (구현·운영 직군 합의)
Playwright + browser-use/Stagehand · 자체 Fargate Spot/Fly Machines · 멀티벤더 프록시
(Decodo+Bright Data 폴백) · GPT-4o-mini/Haiku 폴백(판정 대부분 룰베이스) ·
Postgres(Supabase)+ClickHouse · Cloudflare R2(증거) · Stripe Billing+Unkey(API) ·
Next.js 리더보드 · Inngest 스케줄.
- **단위경제:** 1~5천 사이트/일이 1인 흑자 스윗스팟(프록시 GB가 원가 80% → 2단
  프로브·캐싱으로 GB 70%↓가 마진의 핵심).

## 9. GTM (마케팅 직군)
- **머천트 단면 먼저(12개월), 양면 동시 콜드스타트 금지.**
- 채널: Shopify 앱스토어("Agent Readiness Audit" 앱)·Partners/Unite, DTC 커뮤니티
  (2PM·Lenny's·Shoptalk), 이커머스 에이전시 화이트라벨.
- 콜드아웃리치 후크: **"어제 우리가 ChatGPT/Gemini로 당신 사이트 결제를 시도했고
  N단계에서 깨졌습니다 — 무료 리포트"**(이미 측정한 결과를 미끼로).
- 가격: 머천트 진단 SaaS $200~500/월(엔터 $1~3K), 에이전트 데이터 API는 종량(2차).
- 무료 리더보드 = 데이터 저널리즘 PR + 인바운드 + 권위.

## 10. 14일 검증 (빌드 전 게이트 — `validation/` 하니스 사용)
1. **핵심 전제 직접 측정**: 상위 15곳을 에이전트로 결제 시도 → 단계별 통과율 측정
   (`python -m validation --driver playwright`). 이게 곧 첫 데이터·첫 PR.
   - **합격:** 15곳 중 8곳 이상 도달률 < 70%.
2. **양면 인터뷰**(머천트 5문 / 에이전트 개발자 5문) — 지불 의향.
   - **합격:** 머천트 10곳 중 3곳 이상 "개선 의향 있음".
3. **가짜문 리더보드 랜딩 + 대기자** — 합격: 머천트 10사 또는 대기자 200명.
4. **출시 전 IT 법률 자문 1시간**(필수).
> 하나라도 미달이면 피벗 또는 중단.

## 11. 직군별 95 스코어카드 (원안 → v2)
| 직군 | 원안 | v2 해소 |
|------|:--:|------|
| security | 22 | 무동의 주행·결제제출 폐기, 동의/샌드박스만 |
| implementer | 52 | "완주"→"도달성+실패분해"로 측정 가능화 |
| tester | 54 | 14일 직접측정으로 전제 검증, 단면 선진입 |
| devops | 68 | 동의·캐싱·2단 프로브로 프록시 GB 70%↓ |
| planner | 70 | reachability/completion 2계층 + 증거 + 교차검증 |
| marketer | 71 | 멀티에이전트 중립만 측정·리더보드 권위 선점 |
→ 6직군의 "95로 가는 스펙변경"이 한 방향으로 일치.

## 12. 리스크
- **거인 흡수**(Stripe/Shopify가 당사자 인증 내재화) → 멀티에이전트 중립성으로만 방어.
- **측정 면적 축소**(프로토콜 통일) → "프로토콜 준수"가 아니라 "실제 완주율·실패원인"으로 정의 고정.
- **법적**(주행 합법성) → 동의 기반 + 법률 자문 게이트.
- **단위경제**(프록시 단가 인질) → GB 절감·멀티벤더 헤지.

## 13. 로드맵
- **Phase 0 (2주):** 검증 게이트 통과(위 10).
- **Phase 1 (4~6주):** reachability 측정 + 공개 리더보드 + 머천트 무료→유료 진단(단면).
- **Phase 2 (6~12개월):** 동의 머천트 completion 측정, 방법론 표준화, 권위 선점.
- **Phase 3:** 측정 규모 임계 후 에이전트 데이터 API(2차 수익).
